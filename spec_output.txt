4. Compliance and Non-Advisory Guardrails
4.1 Regulatory Position
In Phase 1, the platform operates as an information portal and requires no Bangladesh Bank licence. The AFC must remain within this boundary. The key constraints are:[^5]
Personal Data Protection Ordinance 2025 (PDPO 2025): Requires explicit, specific, and revocable consent before processing personal data. Financial data is classified as sensitive. The AFC must never persist raw financial inputs from conversational flows to server storage. All loan calculations must remain client-side or in-session only, consistent with the existing privacy architecture.[22][23][^16]
Cyber Security Act 2023: Section 26 prohibits collecting identity information without lawful authority. AFC must not prompt for NID, phone number, or other identity data in Phase 1.[^5]
Information Only Rule: The AFC must not say "you should take this loan" or "this product is best for you." It can say "compared to your current loan, this product has an estimated lower monthly repayment of X taka based on published rates."

9.3 NLP and Language Models

For the intent classifier, a fine-tuned DistilBERT or mBERT on a Bangla financial query dataset is recommended, using the BNLP toolkit for preprocessing (tokenisation, NER, stopword removal for Bangla). The TEXTOIR platform provides open-source tooling for open intent recognition, critical for gracefully handling out-of-scope queries.[48][49]

Conclusion
The AI Financial Companion is not merely a chatbot layer — it is the primary inclusion mechanism for reaching non-privileged users who cannot navigate structured English-language web forms. Its architecture is technically viable using open-source components that align with the existing platform stack. Its guardrails are well-defined and enforceable within the non-advisory product rules. Its rollout path fits squarely within the existing Phase 1A–1C roadmap without displacing any planned deliverable.
The highest-priority action is to begin building the AFC in parallel with the existing tool integration sprint, starting with the text-chat loan assessment flow, which can reuse the credit engine immediately. The voice layer, IVR fallback, and WhatsApp channel are phased additions that progressively close the access gap for Tier 2 and Tier 3 users who represent the majority of the platform's intended beneficiaries.
